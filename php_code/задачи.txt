Описание реализуемого мини-проекта


ЗАДАЧИ
1. Создать возможность авторизации простой (без валидаций e-mail)
2. Создать возможность регистрации нового пользователя
3. Создать возможность регистрации нового администратора

ОБРАЩЕНИЯ
4. Создать страницу с формой для создания обращения
5. Добавить статус для созданного пользователем обращения
ToDo
InProgress
ReadyForReview
Done

6. Список обращений с полями
Task ID
Created date
Updated date
Status
Description

7. Добавить возможность фильтровать и сортировать список в таблице

ЗАДАЧИ - ФРОНТ КЛИЕНТА
8. Создать страницу создания задачи. Поля:
   (модальное окно, либо рядом где-то статичная форма, отдельная таба)
    Список полей при создании задачи:
          Title
          Description
  9. Обмен данными должен быть реализован через AJAX

ПРОМЕЖУТОЧНАЯ ЗАДАЧА: добавить единый CSS ко всем HTML страницам

10. ФРОНТ АДМИНА
   11. Авторизация базовая
   12. Таблица с заполненными обращениями от клиентов и возможность
        фильтрации и сортировки
   13. Просмотр обращения (либо страница, либо модальное окно)

   14. Возможность оставить ответ клиенту и изменить статус
   15. Возможность задать тэги обращению
        (любой набор, к примеру: tech issue, tech question, fatal error, sales question...)
   16. Плюсом будет реализация CRUD страниц для сущеностей: Tags, Statuses

ОБЩИЕ ТРЕБОВАНИЯ
    1. PHP 8+ без использования фреймворков
        Допустимо использование различных библиотек доступных в composer
    2. Используем API подход для обмена данными с фронтом
+    3. Composer с PSR-4 автозагрузкой
    4. БД -  MyS_L, в PHP - PDO.
        Реализуем связи сущностей, которым это необходимо
    5. Фронт - на усмотрение, можно React и Vue и чистый HTML + j_uery
    6. Дизайн не имеет особого значения, главное, чтобы было читаемо
    7. Можно обернуть в Docker образ по желанию

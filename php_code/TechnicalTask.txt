Описание реализуемого мини-проекта

Трекер обращений клиентов, где каждый авторизованный пользователь,
заполнив форму создаёт задачу, которая попадает к администраторам.

Клиент может видеть статус ранее созданного обращения.

Требования к фронту клиента:
+/+  1. Авторизация / Регистрация
+    Базовая (логин, пароль) без валидаций мэйла и так далее
+  2. Список задач в виде таблицы с полями:
+  // СОЗДАТЬ МОДЕЛЬ ИЛИ ОБЪЕКТ "ЗАДАЧА" + ВЫВОДИТЬ ЭТУ ТАБЛИЦУ НА СТРАНИЦЕ
+     ЗАПОЛНЯТЬ ТАБЛИЦУ ИЗ БАЗЫ ДАННЫХ
+        Task ID
+        Created date
+        Updated date
+        Status
+        Description
+      Необходимо иметь возможность фильтровать и сортировать список в таблице
+      Список статусов:
+          ToDo
+          InProgress
+          ReadyForReview
+          Done

+  3. Форма создания задачи
+      (модальное окно, либо рядом где-то статичная форма, отдельная таба)
+      Список полей при создании задачи:
+          Title
+          Description
  4. Обмен данными должен быть реализован через AJAX

Требования к фронту админа:
+   1. Авторизация базовая
+   2. Таблица с заполненными обращениями от клиентов и возможность
+        фильтрации и сортировки
   3. Просмотр обращения (либо страница, либо модальное окно)

   4. Возможность оставить ответ клиенту и изменить статус
   5. Возможность задать тэги обращению
        (любой набор, к примеру: tech issue, tech question, fatal error, sales question...)
   6. Плюсом будет реализация CRUD страниц для сущеностей: Tags, Statuses

Общие Требования
    1. PHP 8+ без использования фреймворков
        Допустимо использование различных библиотек доступных в composer
    2. Используем API подход для обмена данными с фронтом
+    3. Composer с PSR-4 автозагрузкой
+    4. БД -  MySQL, в PHP - PDO.
        Реализуем связи сущностей, которым это необходимо
    5. Фронт - на усмотрение, можно React и Vue и чистый HTML + j_uery
    6. Дизайн не имеет особого значения, главное, чтобы было читаемо
    7. Можно обернуть в Docker образ по желанию

Результат работы предоставить в виде ссылки на GitHub или любой иной аналог.
В крайнем случае архивом с дампом БД и файлами проекта без папки vendor.
